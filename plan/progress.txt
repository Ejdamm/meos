  - Fixed TypeScript errors related to Vite 7 default strict settings (erasableSyntaxOnly, verbatimModuleSyntax).
- **Learnings for future iterations:**
  - The legacy C++ backend (restserver.cpp) implements an XML API at /meos, but the frontend architecture requires a JSON REST API at /api/v1.
  - Avoid TypeScript enums as they are not compatible with 'erasableSyntaxOnly' enabled by default in Vite 7 templates. Use 'const object + type' pattern instead.
  - Always use 'import type' when importing interfaces/types to satisfy 'verbatimModuleSyntax'.
  - Mapping C++ entity fields (from oEvent.h, oRunner.h, etc.) to TypeScript is straightforward but requires careful checking of header files for status codes and optional fields.
