## 2026-03-09 - US-008: Classes page with CRUD
- What was implemented:
  - Created `useCourses.ts` hook for fetching courses.
  - Updated `api/types.ts` to include `StartMethod` enum and `startMethod` field in `Class` interface.
  - Updated `mocks/data.ts` to include `startMethod` in mock classes.
  - Updated `DataTable.tsx` to support `isLoading` prop for better UX.
  - Implemented `ClassesPage.tsx` with full CRUD for classes.
  - Integrated `DataTable`, `FormDialog`, `ConfirmDialog`, `FormInput`, and `FormSelect` on the Classes page.
  - Added `zod` validation for class fields (name, courseId, startMethod, sortOrder).
  - Fixed TypeScript issues with `useMutation` and `zodResolver`.
- **Learnings for future iterations:**
  - `useMutation` in the current implementation only takes one argument for `mutate`. Options must be passed to `useMutation` hook itself, or handled via `async/await` in the caller.
  - `z.coerce.number()` might cause type inference issues with `zodResolver` if not careful; sometimes setting `valueAsNumber: true` in `register` and using `z.number()` is cleaner.
  - `FormSelect` values are always strings; remember to convert back to numbers if needed for the API.
  - `DataTable` is a powerful generic component but needs `isLoading` state to avoid flashing "No data" while fetching.
