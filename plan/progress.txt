## Codebase Patterns
- Use CMake with presets (`CMakePresets.json`) for building.
- Dependencies are managed via `vcpkg` and `vcpkg.json`.
- The modern source code is located in `src/`.
- Executable name is `meos`.
- Redefine common Win32 types (e.g., `SYSTEMTIME`, `DWORD`, `HWND`) in `src/util/win_types.h` for legacy compatibility on Linux.
- Avoid redundant integer conversion overloads (`itos`, `itow`) that overlap on 64-bit platforms (e.g., `unsigned long` vs `uint64_t`).
- For domain object unit tests, ensure `oe->oControlData` (or relevant data container) is initialized.
- Include template implementation files (e.g., `*impl.hpp`) at the end of their respective headers.
- Move shared enums to `domain_header.h` or `src/util/common_enums.h` to avoid circular dependencies between core domain entities.
- Always call `migrate()` after opening the database to ensure the schema is up to date.
- When migrating tightly coupled domain entities, use heavy stubbing in `oEvent.h`, `oClass.h`, etc., to allow incremental migration.
- Always use `getId()` instead of direct `Id` access if the base class or stubs might change.
- Use `narrow()` and `widen()` from `meos_util.h` for easy string conversions during migration.
- Batch fix `stdafx.h` and other Win32-specific headers using `sed` across the migrated files.
- Provide a robust Win32 shim layer in `win_types.h` including file search (`WIN32_FIND_DATA`), string safe functions (`swprintf_s`, `snprintf_s`), and basic GUI types (`RECT`, `COLOR`).
